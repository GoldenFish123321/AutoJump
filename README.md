# 跳一跳 电脑端自动脚本

基于图像识别 + 物理模型的微信跳一跳 PC 端自动脚本。

## 原理

### 识别算法

1. 颜色掩码定位棋子底部中心
2. 从上往下扫描第一块非背景像素，得到下一目标块的顶点 x
3. 等距投影几何（tan30°）推出目标中心，算出像素距离 D
4. 在几何中心附近搜索 F5F5F5 完美跳跃白点：命中则用白点精确校准目标中心，并据此判定上一跳是否完美

### 物理模型（根据游戏源码反推）

#### 符号

| 符号 | 含义 | 单位 |
|------|------|------|
| $h$ | 按压时间 | 秒 (s) |
| $D$ | 屏幕上识别到的跳跃距离 | 像素 (px) |
| $p$ | 实际按压时长 | 毫秒 (ms) |
| $v_z(h)$ | 游戏内水平初速度 | 游戏单位/s |
| $v_y(h)$ | 游戏内竖直初速度 | 游戏单位/s |
| $g$ | 游戏内重力加速度 | 游戏单位/s² |
| $t_{\text{air}}(h)$ | 空中理论飞行时间 | 秒 (s) |
| $x(h)$ | 游戏内 3D 水平移动距离 | 游戏单位 |
| $k$ | 屏幕像素与 3D 距离的比例系数 | px/游戏单位 |
| $T_{\text{stop}}(h)$ | 停稳时间（含飞行+落地动画+场景移动） | 毫秒 (ms) |

#### 源码常量

$$v_{z,\text{inc}} = 70,\quad v_{y,0} = 135,\quad v_{y,\text{inc}} = 15,\quad g = 720$$

#### 速度公式

$$v_z(h) = \min(70h,\ 150)$$

$$v_y(h) = \min(135 + 15h,\ 180)$$

在实测范围 $(h < 1)$ 内不触发上限，简化为：

$$v_z(h) = 70h,\quad v_y(h) = 135 + 15h$$

#### 空中飞行时间

$$t_{\text{air}}(h) = \frac{2v_y(h)}{g} = \frac{2(135 + 15h)}{720} = 0.375 + \frac{h}{24}\ \text{s}$$

换算毫秒：

$$t_{\text{air,ms}}(h) = 375 + 41.67h\ \text{ms}$$

#### 游戏内 3D 水平距离

$$x(h) = v_z(h) \cdot t_{\text{air}}(h) = 70h \cdot \frac{2(135 + 15h)}{720} = 26.25h + \frac{35}{12}h^2$$

#### 屏幕像素与 3D 距离的关系

$$D \approx k \cdot x(h)$$

$k$ 取决于屏幕分辨率/DPI，1080p 下拟合值 $k \approx 17.62$。脚本运行中通过完美跳跃自动校准 $k$。

#### 按压时间反解

已知屏幕距离 $D$，解二次方程求 $h$：

$$\frac{35}{12}h^2 + 26.25h - \frac{D}{k} = 0$$

$$h = \frac{-26.25 + \sqrt{26.25^2 + 4 \cdot \frac{35}{12} \cdot \frac{D}{k}}}{2 \cdot \frac{35}{12}}$$

$$p = 1000h\ \text{ms}$$

#### 停稳时间

$$T_{\text{stop}}(h) \approx t_{\text{air,ms}}(h) + 1102 = 1477 + 41.67h\ \text{ms}$$

在典型按压范围内 $41.67h$ 仅变化几十毫秒，实际可近似为 $T_{\text{stop}} \approx 1.5\text{s}$。

### k 自动校准

每次检测到完美跳跃（下一帧出现 F5F5F5 白点 = 上一跳正中靶心），反算：

$$k_{\text{实测}} = \frac{D_{\text{上跳}}}{x(h_{\text{上跳}})}$$

使用指数移动平均 (EMA) 更新：

$$k_{\text{new}} = \alpha \cdot k_{\text{实测}} + (1 - \alpha) \cdot k_{\text{旧}}$$

其中 $\alpha$ 自适应衰减：

$$\alpha(n) = \frac{0.3}{\sqrt{1 + n/10}}$$

| 样本数 n | α | 效果 |
|----------|-----|------|
| 0~3 | 0.30~0.26 | 快速追踪初始校准 |
| 10 | 0.21 | 适度平滑 |
| 50 | 0.12 | 高度稳定 |
| 100 | 0.09 | 几乎锁定 |

### 自适应按压抖动

对按压时间施加三角分布对称抖动，幅度随校准样本数增加而缩小：

$$r(n) = 0.03 \times \max\left(\frac{1}{\sqrt{n + 1}},\ 0.3\right)$$

$$p_{\text{实际}} = p_{\text{名义}} \times f,\quad f \sim \text{Tri}(1-r,\ 1+r,\ 1.0)$$

| 样本数 n | 抖动幅度 r | 效果 |
|----------|-----------|------|
| 0 | ±3.0% | 充分探索 |
| 3 | ±1.5% | 快速收敛 |
| 10+ | ±0.9% | 锁定下限 |

### 非完美跳微扰

非完美跳后对 $k$ 施加 ±0.3% 随机扰动（不污染 EMA），打破「相同 k → 相同按压 → 反复落偏」的确定性死循环。

## 用法

### 环境

```bash
pip install -r requirements.txt
```

### 第一步：自动识别游戏区域

把游戏右上角、左下角各截一张小图，存为 `corner_tr.png` 和 `corner_bl.png`，放在本目录。建议在分数为 0 的新局下截图，匹配最准。

```bash
python jump_pc.py region
```

脚本全屏截图后用模板匹配定位两个角，反推出游戏矩形，结果存入 `config.json`。匹配预览图存到 `debug/region_match.png`。

### 第二步：测试识别

```bash
python jump_pc.py test
```

截一帧看识别对不对，标注图存到 `debug/test.png`（绿点=棋子落脚、红点=目标中心、品红=完美白点）。

### 第三步：自动跳

```bash
python jump_pc.py run
```

## 运行中热键

| 按键 | 功能 |
|------|------|
| `↑` / `↓` | 增减 k（步长 0.05） |
| `空格` | 暂停 / 继续 |
| `d` | 存一张当前识别 debug 图 |
| `s` | 保存当前 k 到 `config.json` |
| `q` / `Esc` | 退出（自动保存 k） |


## 依赖

- Python 3.x
- opencv-python — 图像识别
- mss — 屏幕截图
- pynput — 鼠标控制与键盘监听
- numpy — 数组运算

## 配置

`config.json` 示例：

```json
{
  "region": [left, top, width, height],
  "phys_k": 17.62
}
```

| 字段 | 说明 |
|------|------|
| `region` | 游戏画面在屏幕上的矩形区域 `[左, 上, 宽, 高]` |
| `phys_k` | 屏幕像素与 3D 距离的比例系数，自动校准 |
